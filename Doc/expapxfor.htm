An Example Forward Mode Sweep

Mathematical Form
Suppose that we use the algorithm ExpApx.hpp to compute  {\rm ExpApx} (x, e) with  x is equal to .5 and  e is equal to .2. For this case, our approximation of the exponential function, and the partial derivative of the approximation, are given by  \[
\begin{array}{rclll}
{\rm ExpApx} (x, e) & = &   1 + x + x^2 / 2  
     & =   1 + .5 + .25 / 2 
     & =   1.625
\\
\partial_x {\rm ExpApx} (x, e) & = &   1 + x 
     & =   1 + .5 
     & =   1.5
\end{array}
\] 

Forward Example
Forward mode uses the ExpApx operation sequence in its original order to compute the partial derivative of all the dependent variables with respect to one independent variable. The operation sequence depends on e, but given the operation sequence, the atomic operations do not depend on e. Thus, the partial derivative with respect to e, of any of the variables in the operation sequence, is zero. We also note that the partial derivatives, with respect to  x , are zero for the following symbols:  \[
     r_0 , s_0 , k_0 , k_1 , k_2 
\]  One forward mode sweep calculates the partial derivative with respect to x (of all the variables) as follows:
Operation    Value    Partial    Value
 a_0 = x  = 0.5  \partial_x a_0 = \partial_x x   = 1
 q_1  = r_0 * a_0   = 0.5  \partial_x q_1 = \partial_x r_0  * a_0 + r_0 * \partial_x a_0 = 1
 r_1  = q_1 / k_0   = 0.5  \partial_x r_1 = \partial_x q_1 / k_0  = 1
 s_1  = s_0 + r_1   = 1.5  \partial_x s_1 = \partial_x s_0 + \partial_x r_1  = 1
 q_2 =  r_1 * a_0  = 0.25  \partial_x q_2 = \partial_x r_1  * a_0 + r_1 * \partial_x a_0  = 1
 r_2 = q_2 / k_1  = 0.125  \partial_x r_2 = \partial_x q_2 / k_1  = 0.5
 s_2   = s_1 + r_2  = 1.625   \partial_x s_2 = \partial_x s_1 + \partial_x r_2  = 1.5
Thus forward mode returns 1.5 as the derivative of the return value of the algorithm for this case.

Verification
The file ExpApxFor.cpp contains a routine which verifies the values computed above. It returns true for success and false for failure.
